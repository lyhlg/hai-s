# Components

## 백엔드 (Node.js / Express)

### 1. Auth Component
- **목적**: 인증/인가 처리
- **책임**:
  - 관리자 로그인 (매장 ID + 사용자명 + 비밀번호)
  - 테이블 태블릿 로그인 (매장 ID + 테이블 번호 + 비밀번호)
  - JWT 토큰 발급/검증
  - 로그인 시도 제한
- **인터페이스**: REST API

### 2. Store Component
- **목적**: 매장 정보 관리
- **책임**:
  - 매장 데이터 조회
  - Seed 데이터 기반 초기 매장/관리자 계정 관리
- **인터페이스**: 내부 서비스 호출

### 3. Table Component
- **목적**: 테이블 및 세션 관리
- **책임**:
  - 테이블 CRUD (번호, 비밀번호)
  - 테이블 세션 라이프사이클 (시작/종료)
  - 세션 ID 생성 및 관리
- **인터페이스**: REST API

### 4. Menu Component
- **목적**: 메뉴 정보 관리
- **책임**:
  - 메뉴 CRUD (메뉴명, 가격, 설명, 카테고리, 이미지 URL, 인기 여부, 품절 여부)
  - 카테고리 관리
  - 메뉴 노출 순서 관리
  - 필드 검증 (필수 필드, 가격 범위)
- **인터페이스**: REST API

### 5. Order Component
- **목적**: 주문 처리 및 관리
- **책임**:
  - 주문 생성 (메뉴 검증 포함)
  - 주문 상태 변경 (대기중/준비중/완료)
  - 주문 삭제
  - 주문 조회 (현재 세션, 과거 이력)
  - 테이블별 총 주문액 계산
- **인터페이스**: REST API

### 6. Settlement Component
- **목적**: 매출 정산
- **책임**:
  - 일매출 집계 (완료된 주문 기준)
  - 날짜별 매출 조회
  - 테이블별 매출 내역
- **인터페이스**: REST API

### 7. SSE Component
- **목적**: 실시간 이벤트 전달
- **책임**:
  - SSE 연결 관리 (고객용, 관리자용)
  - 주문 생성/상태변경 이벤트 브로드캐스트
  - 매장별 이벤트 격리
  - 연결 끊김 시 재연결 지원
- **인터페이스**: SSE Endpoint

## 프론트엔드 - 고객용 (React)

### 8. Customer App
- **목적**: 테이블 고객 주문 인터페이스
- **책임**:
  - 자동 로그인 / 초기 설정
  - 메뉴 조회 (카테고리별, 인기 배지)
  - 장바구니 관리 (로컬 저장)
  - 주문 생성 및 확인
  - 주문 내역 조회 (SSE 실시간 업데이트)

## 프론트엔드 - 관리자용 (React)

### 9. Admin App
- **목적**: 매장 관리자 운영 인터페이스
- **책임**:
  - 매장 로그인
  - 실시간 주문 모니터링 대시보드 (SSE)
  - 테이블 관리 (설정, 세션 종료, 주문 삭제)
  - 메뉴 관리 (CRUD, 순서, 인기 지정)
  - 과거 주문 내역 조회
  - 일매출 정산
