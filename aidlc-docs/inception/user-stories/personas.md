# User Personas

---

## 고객 페르소나

### Persona 1: 김지수 (기술 친숙 고객)
- **역할**: 테이블 고객 (20~30대)
- **목표**: 빠르고 효율적으로 주문 완료
- **Pain Points**: 직원 호출 대기 시간, 메뉴판 넘기는 번거로움
- **기술 숙련도**: 높음 - 스마트폰/태블릿 일상적 사용, 키오스크 익숙

### Persona 2: 박영숙 (기술 미숙 고객)
- **역할**: 테이블 고객 (50~60대)
- **목표**: 원하는 메뉴를 실수 없이 주문
- **Pain Points**: 작은 글씨, 복잡한 UI, 실수로 잘못 주문할까 불안
- **기술 숙련도**: 낮음 - 스마트폰 기본 사용 가능, 키오스크 경험 적음

---

## 관리자 페르소나

### Persona 3: 이민호 (매장 관리자)
- **역할**: 매장 사장님 겸 운영자
- **목표**: 주문을 놓치지 않고 매장을 효율적으로 운영
- **Pain Points**: 바쁜 시간대 주문 누락, 테이블별 주문 현황 파악 어려움, 정산 실수
- **기술 숙련도**: 중간 - PC/태블릿 기본 사용 가능, 복잡한 시스템은 부담

---

## 확장 고려사항: 관리자 권한 분리

향후 매장 규모 확대 시 아래와 같이 페르소나 분리 가능:

| 구분 | 매장 사장님 | 매장 직원 |
|------|-----------|----------|
| 메뉴 관리 | ✅ CRUD 전체 | ❌ 조회만 |
| 테이블 세션 종료 | ✅ | ✅ |
| 주문 삭제 | ✅ | ⚠️ 승인 필요 |
| 과거 내역 조회 | ✅ 전체 | ✅ 당일만 |
| 매장 설정 변경 | ✅ | ❌ |

현재 MVP에서는 단일 "매장 관리자" 역할로 통합하되, 데이터 모델에 role 필드를 포함하여 확장 가능하도록 설계.
